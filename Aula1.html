Atividades Práticas de Autenticação e Autorização no SQL Server

Atividade 1: Verificar o Modo de Autenticação Atual

Tipos de Autenticação no SQL Server

  • Autenticação do Windows: Utiliza as credenciais do Windows para autenticar usuários. É considerada mais segura porque aproveita os mecanismos de segurança do Windows, como políticas de senha e autenticação Kerberos.
  • Autenticação do SQL Server (Mista): Permite que o SQL Server gerencie as credenciais de login. Isso é útil em ambientes onde usuários não têm contas do Windows ou quando é necessário acesso a partir de sistemas não-Windows.

Importância de Utilizar Cada Tipo de Autenticação

  • Autenticação do Windows:
    • Segurança: Aproveita as políticas de segurança do Windows, como complexidade de senha e bloqueio de conta.
    • Gerenciamento Centralizado: Facilita o gerenciamento de usuários e permissões através do Active Directory.
    • Autenticação Integrada: Usuários não precisam fornecer credenciais adicionais ao acessar o SQL Server, pois a autenticação é feita automaticamente.
  • Autenticação do SQL Server (Mista):
    • Flexibilidade: Permite que usuários sem contas do Windows acessem o SQL Server.
    • Compatibilidade: Necessária para aplicativos que não suportam autenticação do Windows.
    • Isolamento: Útil em ambientes onde o SQL Server precisa ser isolado do domínio do Windows por razões de segurança ou arquitetura.

Verificar o Modo de Autenticação Atual

SELECT SERVERPROPERTY('IsIntegratedSecurityOnly') AS [AuthMode];

Resultado:

  • 1: O SQL Server está configurado para usar apenas autenticação do Windows.
  • 0: O SQL Server está configurado para usar autenticação mista (Windows e SQL Server).

Atividade 2: Criar um Login e Usuário

Passo 1: Criar um Login de SQL Server

CREATE LOGIN MeuLoginComSenha 
WITH PASSWORD = 'SenhaForte123!';

Explicação:

  • CREATE LOGIN: Este comando cria um novo login no SQL Server.
  • MeuLoginComSenha: Nome do login que estamos criando.
  • WITH PASSWORD: Define a senha para o login. No exemplo, a senha é 'SenhaForte123!'.
  • Importância: Criar logins permite que usuários acessem o SQL Server com credenciais específicas. É uma prática comum em ambientes onde diferentes usuários precisam acessar o banco de dados com diferentes níveis de permissão.

Passo 2: Criar um Usuário de Banco de Dados

USE MeuBancoDeDados;
CREATE USER MeuUsuario 
FOR LOGIN MeuLoginComSenha;

Explicação:

  • USE MeuBancoDeDados: Este comando seleciona o banco de dados `MeuBancoDeDados` para que os comandos subsequentes sejam executados nele.
  • CREATE USER: Este comando cria um novo usuário no banco de dados selecionado.
  • MeuUsuario: Nome do usuário que estamos criando.
  • FOR LOGIN MeuLoginComSenha: Associa o usuário `MeuUsuario` ao login `MeuLoginComSenha` criado anteriormente.
  • Importância: Criar usuários no banco de dados permite que logins tenham acesso a recursos específicos dentro de um banco de dados. Isso é crucial para gerenciar permissões e garantir que os usuários só possam acessar os dados e executar as operações que lhes foram explicitamente concedidas.

Atividade 3: Conceder Permissões

Conceder Permissões de Leitura

USE MeuBancoDeDados;
GRANT SELECT ON SCHEMA::dbo TO MeuUsuario;

Resultado:

  • O usuário `MeuUsuario` terá permissão para executar consultas SELECT no esquema `dbo` do banco de dados `MeuBancoDeDados`.

Conceder Permissões de Escrita

USE MeuBancoDeDados;
GRANT INSERT, UPDATE, DELETE ON SCHEMA::dbo TO MeuUsuario;

Resultado:

  • O usuário `MeuUsuario` terá permissão para executar operações INSERT, UPDATE e DELETE no esquema `dbo` do banco de dados `MeuBancoDeDados`.

Atividade 4: Verificar Permissões

Verificar Permissões de um Usuário

USE MeuBancoDeDados;
EXEC sp_helprotect @username = 'MeuUsuario';

Resultado: Uma lista de permissões concedidas ao usuário `MeuUsuario` será exibida, detalhando quais ações ele pode realizar no banco de dados `MeuBancoDeDados`.

Atividade 5: Criar e Utilizar Papéis (Roles)

Passo 1: Criar um Papel de Banco de Dados

USE MeuBancoDeDados;
CREATE ROLE MeuPapel;

Explicação:

  • CREATE ROLE: Este comando cria um novo papel (role) no banco de dados.
  • MeuPapel: Nome do papel que estamos criando.
  • Importância: Papéis permitem agrupar permissões e atribuí-las a múltiplos usuários, facilitando o gerenciamento de permissões.

Passo 2: Atribuir Permissões ao Papel

USE MeuBancoDeDados;
GRANT SELECT, INSERT, UPDATE, DELETE ON SCHEMA::dbo TO MeuPapel;

Explicação:

  • GRANT: Este comando concede permissões ao papel.
  • SELECT, INSERT, UPDATE, DELETE: Permissões que estamos concedendo.
